유휴 베어메탈 서버의 전력을 0W로 — 이벤트 기반 물리 노드 자동 전원 제어
Kubernetes v1.29 Bare-Metal 3-Node Cluster 2026.02 1인 설계 및 구현
Problem

데이터센터 전력의 30%는 좀비 서버에서 낭비됩니다

퍼블릭 클라우드는 서버리스로 해결했지만, 온프레미스 베어메탈 환경에서는 트래픽과 무관하게 서버가 24시간 가동됩니다. 유휴 서버 1대의 대기 전력은 약 300W, 연간 2,628kWh를 소비합니다.

Solution

트래픽이 없으면 물리 서버의 전원을 실제로 차단합니다

Kafka의 버퍼링 + KEDA의 이벤트 기반 스케일링 + 커스텀 컨트롤러의 물리 노드 전원 제어를 결합하여, 유휴 상태의 Worker Node 전력을 0W로 만듭니다. 요청이 들어오면 Wake-on-LAN으로 수 초 내 부팅합니다.
