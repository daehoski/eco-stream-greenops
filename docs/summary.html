프로젝트 요약서 1

개요
프로젝트명 Eco-Kube: GreenOps 인프라 오케스트레이터 구분 개인 / 팀
프로젝트 목적 이벤트 기반 서버리스 아키텍처와 전원 관리 컨트롤러를 통한 클러스터 에너지 효율 최적화
프로젝트 기대효과 불필요한 컴퓨팅 자원 대기 전력 감소, 운영 비용 절감 및 탄소 배출 저감 효과
담당업무(팀프로젝트시) 인프라 구축 및 GreenOps 로직 구현 기간 2026.02.11 ~ 2026.03.03
세부내용
구성 및 작성 내용 1
GreenOps 통합 아키텍처 (Smart Power Control)
평소에는 서버를 재워두고(Sleep), 작업 요청이 들어오면 실시간으로 깨우는(Wake) 지능형 인프라 제어 통합 설계.
4단계 핵심 프로세스 시각화 (Architecture)
(작업 요청 ➔ 파드 대기 ➔ 깨우기 신호 ➔ 작업 시작!)
구성 및 작성 내용 2
파드 대기 (노드 없음) & 트리거 감지
Kafka Lag을 감시하여 파드를 자동 생성하고, 노드가 없을 때 발생하는 'Pending' 상태를 감지하여 부팅 신호를 전송하는 로직 구현.
KEDA Scaling & Pending
(kubectl get pods 시연 화면)
ScaledObject Manifest
(기술적 설정 요약)
구성 및 작성 내용 3
물리 서버 원격 부팅 (WOL & SSH)
GreenOps Controller가 Wake-on-LAN 패킷으로 노드를 깨우고, 인프라 합류를 자동화. 작업 완료 시 SSH Shutdown으로 전력 100% 차단.
Node Join & Ready Status
(kubectl get nodes 화면)
Controller Log Snippet
(WOL & SSH Shutdown 로그)
구성 및 작성 내용 4
통합 모니터링 컨트롤 센터 배포
Grafana 지능형 대시보드를 임베딩하여 인프라의 에너지 소비 효율과 실시간 파드 상태를 운영자가 시각적으로 파악할 수 있도록 구현.
Control Center Dashboard
(메인 웹 화면 전체)
Grafana Metrics Graph
